# 遗传算法
## 遗传算法（Genetic Algorithm，简称 GA）是一种模仿大自然生物进化的计算机算法，用来寻找问题的最优解。

## 首先需要将要解决的问题映射为一个数学问题————即数学建模
## 基因与染色体
这个问题的一个可行的解被称为**一条染色体**

*一个可行的解一般有多个元素组成* 则每一个元素就被称为一个**基因**

## 适应度函数
**则优劣汰**
*如何衡量一个染色体的优劣？*
**————由适应度函数完成**
算法运行过程中进行N此迭代 每次迭代都会产生若干染色体 
**适应度函数会给本次迭代生成的所有染色体打个分 将适应度低的染色体淘汰掉 从而经过若干次迭代后染色体的质量越来越好**

**把问题抽象为可以进行交叉或者变异操作的解**
## 整体的流程
┌─────────────────────────────────────────────────────────────┐
│  1. 初始化种群                                                │
│     随机生成 N 个个体（解），每个个体代表一组决策变量。         │
└───────────────────────────┬─────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  2. 适应度评价                                                │
│     计算每个个体的适应度（目标函数值），越优的个体适应度越高。   │
└───────────────────────────┬─────────────────────────────────┘
                            ▼
        ┌───────────────────┴─────────────────────────────────┐
        │          是否满足终止条件？             │
        │ （达到最大代数 / 精度要求 / 收敛停滞）  │
        └───────────────┬───────────┬───────────┘
                        │ 是        │ 否
                        ▼           ▼
                  【输出最优解】   ┌───────────────────────────┐
                                  │  3. 选择（Selection）      │
                                  │     根据适应度挑选父母个体。 │
                                  └───────────────┬───────────┘
                                                  ▼
                                  ┌───────────────────────────┐
                                  │  4. 交叉（Crossover）      │
                                  │     父母交换部分变量生成子代。│
                                  └───────────────┬───────────┘
                                                  ▼
                                  ┌───────────────────────────┐
                                  │  5. 变异（Mutation）       │
                                  │     以一定概率随机扰动子代基因。│
                                  └───────────────┬───────────┘
                                                  │
                                                  └─────────────► 返回步骤2

## 交叉
遗传算法每一次迭代被称为**进化**
**进化的过程**
需要两条染色体 将两条染色体的某个位置切断 再将切下的拼接在一起
*如何选择染色体呢* 轮盘赌算法
## 轮盘赌算法
**核心思想：适应度越高的个体，被选中的概率越大（但非确定性选择，保持多样性）**



## 变异
染色体位点突变

变量的取值范围由 lb 和 ub 约束：

`lb = [18, 60];   % T_air 下限18℃，fan_speed 下限60%`
`ub = [26, 100];  % T_air 上限26℃，fan_speed 上限100%`

 ## optimize_ga.m 中，使用遗传算法的核心目的是：自动搜索出最优的空调温度 T_air 和风机转速 fan_speed，使得数据中心的每小时电费最小，同时保证机柜入口平均温度不超过 27℃。





































































































# obj = @(x) calc_cost(x(1), x(2), load_fixed);
1. 代码结构与语法
@(x)：表示这是一个匿名函数，接受单个输入参数 x
calc_cost(...)：调用预先定义的能耗计算函数
x(1) 和 x(2)：从输入向量 x 中提取的两个关键参数
load_fixed：一个预定义的固定变量，代表服务器负载水平
2. 参数含义与工作原理
## x(1)：代表空调设定温度（单位：℃）
优化过程中会尝试不同温度值（如20℃、22℃、24℃等）
温度越高，空调制冷功耗通常越低，但需确保设备不超温

## x(2)：代表风机转速（单位：百分比）
优化过程中会调整转速值（如60%、75%、90%等）
转速越低，风机功耗越小，但可能影响散热效果

**obj = @(x)调用流程**
1. 初始定义
此时仅定义了目标函数的结构
不会立即执行calc_cost函数
相当于告诉MATLAB："当需要计算目标值时，使用这个方式调用calc_cost"
2. 遗传算法启动时的调用
# 当执行到此处时候
``` matlab
[x_opt, fval] = ga(obj, 2, [], [], [], [], lb, ub, nonlcon, options);
```
遗传算法开始工作
第一次调用obj：评估初始种群中所有个体的适应度
假设种群大小为50，则第一代会调用50次obj，每次传入不同的x值
